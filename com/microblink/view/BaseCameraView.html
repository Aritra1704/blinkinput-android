    public abstract class BaseCameraView
extends android.view.ViewGroup
    A simple wrapper around a Camera and a SurfaceView that renders a centered preview of the Camera to the surface. We need to center the SurfaceView because not all devices have cameras that support preview sizes at the same aspect ratio as the device's display.

      • setUseTextureViewForCameraDisplay

        public final void setUseTextureViewForCameraDisplay(boolean forceTextureView)
        In order to display camera preview, camera view must use one of two android's view's that are capable of displaying camera preview video stream. When using Camera2 API on Lollipop devices that support it, TextureView will always be used, regardless of this setting, however if legacy camera API is used, SurfaceView will be preferred because it is faster. Sometimes, when you create complex UI on top of camera screen, animations or UI may render incorrectly on some devices when using SurfaceView for displaying camera preview. In that case, you can call this method with parameter 'true' which will ensure TextureView to be used for displaying camera preview rather than SurfaceView. TextureView is slower, but is more UI friendly so it allows creation of more complex UI on top of camera preview. Please note that TextureView requires hardware acceleration to be enabled in your app and on the device. TextureView requires API level 14 (on older API levels, this setting will be ignored and SurfaceView will be used).
        Parameters:
        forceTextureView - Whether TextureView should be used even for legacy camera API.
